# BUG-009: PetSprite inline styles cause performance issues

**Severity:** Medium
**Status:** Fixed
**Found by:** audit
**Assigned to:** generalist-1
**Created:** 2025-12-25 15:39

## Description
PetSprite.tsx injects <style> tag with keyframes on every render

Issue: 
- AnimationStyles component (lines 14-198) creates inline <style> tag
- Gets injected multiple times when multiple pets render
- ~185 lines of CSS duplicated in DOM

Location: src/components/pets/PetSprite.tsx lines 14-198

Fix:
- Extract animations to src/index.css OR
- Use CSS modules OR  
- Create single global injection with useEffect + flag

Impact: Performance degradation with multiple pets on screen

# BUG-010: Inconsistent touch target sizes for mobile

**Severity:** Medium
**Status:** Fixed
**Found by:** audit
**Assigned to:** generalist-2
**Created:** 2025-12-25 15:39

## Description
Touch targets not consistently sized for mobile accessibility

Issue: Only one touch-friendly rule at line 838-844 in index.css
WCAG requires minimum 44x44px tap targets for accessibility

Current: .touch-manipulation class exists but not applied consistently

Files to check:
- Header.tsx - icon buttons should be 44x44px minimum
- ActionButtons.tsx - all buttons need proper sizing
- GachaModal.tsx - modal controls
- All interactive elements

Fix:
- Audit all <button> and interactive elements
- Ensure min-width: 44px, min-height: 44px
- Add touch-manipulation class or enforce via Tailwind classes
- Test on real mobile device (375px viewport)

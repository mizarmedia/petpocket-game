# BUG-022: Missing ARIA labels in GamesMenu component

**Severity:** Medium
**Status:** Fixed
**Found by:** audit
**Assigned to:** fixer-2
**Created:** 2025-12-25 15:45

## Description
The games menu modal has multiple buttons without aria-label attributes, affecting screen reader accessibility.

Buttons missing ARIA labels:
1. Close button ("X") - no aria-label
2. "PLAY" button for Treat Catcher game - no aria-label
3. "PLAY" button for Memory Match game - no aria-label

Expected: All buttons should have descriptive aria-label attributes
Actual: Modal buttons lack accessibility labels

Example fixes needed:
- Close button: aria-label="Close games menu"
- Treat Catcher PLAY: aria-label="Play Treat Catcher game"
- Memory Match PLAY: aria-label="Play Memory Match game"

Impact: Screen reader users cannot identify button purposes in games menu
File: src/components/games/GamesMenu.tsx
Severity: Medium - Accessibility gap
Related: Same pattern as header buttons (which were fixed)

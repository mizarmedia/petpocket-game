# BUG-008: Missing prefers-reduced-motion accessibility support

**Severity:** High
**Status:** Fixed
**Found by:** audit
**Assigned to:** generalist-2
**Created:** 2025-12-25 15:39

## Description
No @media (prefers-reduced-motion: reduce) support anywhere in CSS despite extensive animations.

Issue: Users with motion sensitivity settings will experience all animations
Required: Respect user's motion preferences

Location: src/index.css

Fix needed:
- Add @media (prefers-reduced-motion: reduce) rule
- Disable or reduce animations for users with this preference
- Apply to all keyframe animations (bounce, wiggle, float, sparkle, etc.)

Reference: AUDIT_REPORT.md mentions reduced motion support needed
WCAG 2.1 Level AA requirement

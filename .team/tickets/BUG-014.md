# BUG-014: StatBars use hardcoded gradients instead of CSS classes

**Severity:** Medium
**Status:** Open
**Found by:** audit
**Assigned to:**
**Created:** 2025-12-25 15:41

## Description
Stat bars duplicate gradient definitions instead of using existing CSS

Duplication:
- StatBars.tsx lines 94-124: Hardcoded linear-gradient strings
- index.css lines 262-273: .stat-hunger, .stat-happiness, etc. classes

Issue:
- Theme inconsistency - can't update all gradients from one place
- Harder to maintain
- CSS classes exist but aren't used

Location: src/components/StatBars.tsx lines 94-124

Fix: Use the existing CSS classes from index.css
- Remove colorClass prop (unused)
- Apply .stat-hunger, .stat-happiness, .stat-energy, .stat-cleanliness classes
- Remove hardcoded gradient strings
